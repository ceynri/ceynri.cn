# 前后端交互知识采集规范

前后端分离项目中，前端不感知的后台逻辑是知识库中**最有价值**的内容之一——AI 无法通过读前端代码得知后台的行为。本文档提供前端视角下采集后台相关知识的具体模板。

**适用边界**：本文档模板适用于记录**离散字段 / 枚举值 / 单点后台行为**这类"点状"信息；涉及多字段交织、业务流设计决策的"面状"内容，用自然语言叙述更连贯，不必强套模板。

## 筛选原则：只记录核心字段

**不需要记录接口的所有字段**。只有满足以下条件才值得进入知识库：

- **前端无法自行推断其含义或取值范围** — 如订单状态文案（后台下发，前端只做展示）
- **字段值由后台业务逻辑决定，前端不可控** — 如后台根据身份、灰度策略动态下发不同内容
- **字段取值会影响前端分支逻辑** — 如某个 type 值决定 UI 形态

**不需要记录的**：前端自己构造的请求参数、从字段名就能直接理解的字段（`productId`、`price`）、纯透传不消费的字段。

## 后台下发字段模板

> 这里要记录的是**代码里读不到的后台行为信息**（字段的完整取值集合、触发条件、信息来源），不是复述字段在前端代码里如何使用、如何流转——后者可以通过读代码获知，属于 writing-guidelines 明确排除的"代码直接映射"。

```markdown
### [字段名]（来自 [CGI 接口名]）

- **含义**：字段的业务含义
- **可能的值**：列举**所有已知**的值，不只是当前用到的
  | 值 | 含义 | 触发条件 |
  |----|------|---------|
- **信息来源**：`后台文档`（最可信）/ `开发者口述`（可能遗漏）/ `代码推断`（需验证）
- **前端使用位置**：前端在哪里消费此字段
```

## 后台隐藏逻辑

完全由后台控制、前端只展示返回结果的业务逻辑（身份差异化文案、灰度策略、字段值随订单生命周期变化等），用 blockquote 显著标注：

```markdown
> ⚠️ 后台隐藏逻辑（信息来源：[来源]）
>
> [描述后台的行为]
```
